The Problem with Manual Bank Reconciliation

Bank reconciliation is one of the most time-consuming tasks in accountancy. For practices handling multiple clients, the volume multiplies fast. The firm we worked with had a specific challenge: a high volume of transactions across multiple accounts, all needing to be matched, categorised, and reconciled against bank statements before month-end reporting.

Their team was spending two full working days per month on this task. That is roughly 16 hours of skilled staff time on work that is repetitive, rule-based, and automatable. Manual reconciliation also carries real risk beyond the time cost — tired staff miss mismatches, small errors compound, and the closer you are to a deadline the less time there is to investigate discrepancies.

For UK accountancy practices, Making Tax Digital (MTD) is adding pressure to keep records cleaner and more current. The case for automation is straightforward: this is skilled work being consumed by a task software can handle.

The Workflow: Make.com + Xero API + Google Sheets

The automation runs across three tools. No custom code. No enterprise software. The complete build took under two weeks.

01

Pull unreconciled transactions from Xero

Make.com connects to the Xero API using OAuth 2.0. A scheduled scenario runs nightly and pulls all new bank transactions from the client's connected accounts. The Xero Bank Transactions endpoint returns each transaction with its date, amount, payee, reference, and status — filtered for UNRECONCILED records only.

02

Match transactions against the bank statement feed

Make.com iterates over each unreconciled transaction and attempts to match it against open statement lines using three criteria: amount (exact match), date (within a three-day window), and payee reference (fuzzy match).

The first version used only amount and date matching — that produced a 60% auto-reconciliation rate. Adding the fuzzy payee reference check pushed it to 85%.

03

Auto-reconcile confirmed matches in Xero

For transactions matched with high confidence, Make.com calls the Xero API to reconcile them automatically via the PUT BankTransactions endpoint with IsReconciled: true. For this client, approximately 85% of transactions — around 68,000 records — met the auto-reconcile threshold in a single overnight run.

04

Route exceptions to a Google Sheet for review

Transactions that do not meet the matching threshold are written to a Google Sheet. Each row includes the transaction details, the closest match found, and the reason it was flagged: amount discrepancy, date gap, or missing payee reference. A member of staff reviews this sheet each morning — instead of working through 80,000 records manually, they are looking at an exceptions list already sorted by confidence score.

05

Log, summarise, and alert

Each nightly run writes a summary to a second Google Sheet: total transactions processed, number auto-reconciled, number flagged, and any API errors. A Make.com email module sends this summary to the practice manager each morning. If the API returns an error or the match rate drops below a configured threshold, an alert fires to a Slack channel. Silent on success, alert on failure.

Frequently Asked Questions

Does this work with all UK banks connected to Xero?

It works with any bank that provides a live feed to Xero — Barclays, HSBC, Lloyds, NatWest, Starling, and others. If the client's bank provides a statement feed in Xero, the automation can process those transactions.

What happens if the Xero API goes down or changes?

Make.com handles API failures by retrying failed operations and logging errors. The nightly run flags any API issues in the morning summary email and fires a Slack alert. The practice can reconcile manually in Xero exactly as before — the automation is additive, it does not replace the underlying Xero workflow.

Where is client financial data stored?

The Google Sheet is owned by the practice and sits within their own Google Workspace. Make.com does not store transaction data beyond the duration of the scenario run. The Xero connection uses OAuth 2.0 — no credentials held in the automation. All data stays within systems the practice already controls.
