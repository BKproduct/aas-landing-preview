# Просмотр лендинга через GitHub (шаги под ваш контроль)

Эта папка — **самодостаточный** набор для **GitHub Pages**: один `index.html` + видео/постер/логотип рядом.

## Шаг 1 — Положить файлы в ваш репозиторий

**Вариант A (рекомендуется):** в корне вашего репозитория создайте папку `docs/` и **скопируйте туда всё содержимое** этой папки `github-pages-bundle/` (чтобы получилось `docs/index.html`, `docs/logo.jpg`, …).

**Вариант B:** скопируйте файлы в **корень** репозитория (тогда в настройках Pages выберите branch `/ (root)` вместо `/docs`).

## Шаг 2 — Включить GitHub Pages

1. На GitHub: **Settings** → **Pages**  
2. **Build and deployment** → Source: **Deploy from a branch**  
3. Branch: **main** (или ваша основная ветка)  
4. Folder: **`/docs`** (если положили в `docs/`) или **`/ (root)`**  
5. Сохраните. Через 1–3 минуты появится зелёная подсказка с URL.

## Шаг 3 — Ссылка, которую вы откроете в браузере

Шаблон:

```text
https://<ВАШ_GITHUB_ЛОГИН>.github.io/<ИМЯ_РЕПОЗИТОРИЯ>/
```

Пример: логин `BKproduct`, репо `aas-landing-preview` →  
`https://bkproduct.github.io/aas-landing-preview/`

Если репо **приватный**, GitHub Pages для free org может быть недоступен — тогда либо **публичный** репо только для превью, либо деплой на **Vercel**.

## Шаг 4 — Анимация / Seedance

Сейчас в превью подключены **`hero-loop.mp4`** и **`hero-poster.jpg`** (относительные пути). Замените файлы в `docs/` (или в корне) своим **Seedance loop** — перезагрузите страницу с сбросом кэша (`?v=2`).

Подробный промпт: в основном проекте `docs/SEEDANCE_HERO_PROMPT.md`.

## Примечание

Это **статическая копия** лендинга для быстрого просмотра. Основной продуктовый код остаётся в Next.js (`aas-landing/`).
